## Deploy to Vercel

Here we outline the process of forking the Flexible DAO Interfaces template and creating your own DAO site powered by your newly deployed onchain theming contract.

<Steps>
### Start 
Begin by clicking the button below and connecting to Vercel via GitHub.

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fvercel%2Fnext.js%2Ftree%2Fcanary%2Fexamples%2Fhello-world)

<Callout>
  If you don’t have a GitHub account, follow our Getting Started with GitHub tutorial.
</Callout>

### Create your new repository 
Create your new repository which will automatically be titled identically to [your DAO Name?]. 
> Make the repository public so we and others can see what you’re building!

### Configure environment variables
Supply the requested environment variables as described below.

| Name                 | Value                                                                                   |
| -------------------- | -------------------------------------------------------------------------------------------------|
| NEXT_PUBLIC_CHAIN_ID | The chain identifier of the network your DAO is deployed to. Enter 5 for the Göerli test network.|
| NEXT_PUBLIC_DAO_ADDRESS | -|
| NEXT_PUBLIC_ACCESS_TOKEN | The access token for website maintainers within the DAO. |
| NEXT_PUBLIC_ONCHAIN_THEMING_CONTRACT | The address of your onchain theming contract that you deployed in the site should currently be copied to your clipboard. If it is not, return to Göerli Etherscan to retrieve it!|
| NEXT_PUBLIC_ALCHEMY_KEY_GOERLI | The API key from your Alchemy dashboard. |

<Callout>
  For detailed instructions on how to retrieve an Alchemy API Key, click here.
</Callout>

### Build
Following the configuration of your environment variables, you’re ready to deploy your project to Vercel! When your project finishes building, you’ll be redirected to a success screen and prompted to continue to your Vercel dashboard.
</Steps>
